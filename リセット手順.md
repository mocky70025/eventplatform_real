# リセット手順

## 🎯 目標
**すべての外部サービス（Supabase、Vercel）を削除し、まっさらな状態から始める**

---

## ⚠️ 注意事項

- **ローカルのコードは保持されます**（Gitリポジトリは残ります）
- **Supabaseプロジェクトを削除すると、すべてのデータが失われます**
- **Vercelプロジェクトを削除すると、デプロイされたアプリが削除されます**
- **削除後は復元できません**

---

## 📋 リセット手順

### 1. Vercelプロジェクトの削除

#### 削除するプロジェクト
1. `store-web`
2. `store-liff`
3. `organizer-web`
4. `admin`

#### 削除手順

1. **Vercel Dashboardにアクセス**
   - https://vercel.com/dashboard

2. **各プロジェクトを削除**
   - プロジェクトを選択
   - 「Settings」→「General」
   - 最下部の「Delete Project」をクリック
   - プロジェクト名を入力して確認

3. **すべてのプロジェクトを削除**
   - 上記の手順を4つのプロジェクトすべてで実行

---

### 2. Supabaseプロジェクトの削除

#### 削除手順

1. **Supabase Dashboardにアクセス**
   - https://supabase.com/dashboard

2. **プロジェクトを選択**
   - 現在のプロジェクト（例: `xszkbfwqtwpsfnwdfjak`）を選択

3. **プロジェクトを削除**
   - 「Settings」→「General」
   - 最下部の「Delete Project」をクリック
   - プロジェクト名を入力して確認

---

### 3. LINE Developers Consoleの設定（オプション）

#### 削除するか、そのままにするか選択

**オプション1: 削除する**
- LINE Developers Consoleにアクセス
- チャネルを削除

**オプション2: そのままにする**
- 後で新しいSupabase/Vercelプロジェクトを設定する際に再利用可能

---

### 4. ローカル環境の整理（オプション）

#### 環境変数ファイルの整理

以下のファイルを確認・削除（必要に応じて）：
- `.env.local`
- `.env.development.local`
- `.env.production.local`

**注意**: これらのファイルは通常 `.gitignore` に含まれているため、Gitには含まれていません。

---

### 5. Gitリポジトリの確認

#### 現在の状態を確認

```bash
# 現在のブランチとコミットを確認
git status
git log --oneline -10
```

#### 必要に応じて、新しいブランチを作成

```bash
# 新しいブランチを作成（オプション）
git checkout -b fresh-start
```

---

## ✅ リセット完了の確認

### 確認項目

- [ ] Vercelのすべてのプロジェクトが削除された
- [ ] Supabaseプロジェクトが削除された
- [ ] ローカルのコードは保持されている
- [ ] Gitリポジトリは保持されている

---

## 🚀 次のステップ

リセット完了後、以下の手順で再セットアップできます：

1. **新しいSupabaseプロジェクトを作成**
2. **データベーススキーマを適用**
3. **新しいVercelプロジェクトを作成**
4. **環境変数を設定**
5. **デプロイ**

---

## 📝 メモ

### 削除前にバックアップが必要な場合

以下の情報をメモしておくことをおすすめします：

- **Supabase**
  - Project URL
  - Anon Key
  - Service Role Key（もし使用していた場合）

- **Vercel**
  - プロジェクトURL
  - 環境変数のリスト

- **LINE Developers**
  - Channel ID
  - Channel Secret
  - LIFF App ID

---

## ⚠️ 最終確認

削除を実行する前に、以下を確認してください：

1. **本当に削除して良いか？**
   - データのバックアップは必要ないか？
   - 後で必要になる情報はないか？

2. **コードは保持されるか？**
   - ローカルのコードは削除されません
   - Gitリポジトリも保持されます

3. **削除後の計画は？**
   - すぐに再セットアップするか？
   - 別のアプローチを試すか？

---

削除を実行する準備ができたら、上記の手順に従って進めてください。
