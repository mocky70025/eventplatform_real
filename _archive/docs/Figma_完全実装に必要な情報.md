# Figma完全実装に必要な情報

## 私が完全に実装するために必要な情報

### ✅ 必須情報（これがあれば100%再現可能）

1. **Figma URL（node-id付き）**
   ```
   https://figma.com/design/[fileKey]/[fileName]?node-id=[nodeId]&m=dev
   ```
   - 例: `https://figma.com/design/XxOyYjBZdBDkaAbqjpbJJ1/...?node-id=488-473&m=dev`
   - `m=dev` があるとDev Modeの情報も取得できます（推奨）

2. **画面の説明（オプション）**
   - 例：「出店者プロフィール確認画面」
   - あると実装の意図がわかりやすい

## 私が取得できる情報（Figma URLから自動取得）

### 1. レイアウト情報
- ✅ 画面サイズ（width, height）
- ✅ 各要素の位置（x, y座標）
- ✅ 各要素のサイズ（width, height）
- ✅ padding, margin, gap

### 2. スタイル情報
- ✅ 背景色・グラデーション（HEX, RGBA値）
- ✅ テキスト色（HEX値）
- ✅ ボーダー色・太さ
- ✅ フォント（ファミリー、サイズ、ウェイト、スタイル）
- ✅ 角丸（border-radius）
- ✅ シャドウ（box-shadow）
- ✅ 透明度（opacity）

### 3. コンテンツ情報
- ✅ テキスト内容
- ✅ 画像URL（7日間有効）
- ✅ アイコン（SVGとして取得可能）

### 4. レイアウト構造
- ✅ 要素の階層関係
- ✅ flex/gridレイアウト設定

## 完全実装のためのベストプラクティス

### 方法1: Figma URLのみ（最も簡単・推奨）

```
「この画面を実装してください」
https://figma.com/design/...?node-id=488-473&m=dev
```

**これだけで以下の情報を取得します**：
1. デザインコンテキスト（get_design_context）
   - レイアウト、スタイル、コンテンツの全て
2. スクリーンショット（get_screenshot）
   - 視覚的な確認用
3. メタデータ（get_metadata）
   - 構造情報

### 方法2: 複数画面を一度に提供

```
「以下の画面を実装してください」
1. ログイン画面: https://figma.com/...?node-id=488-465&m=dev
2. 登録画面: https://figma.com/...?node-id=488-466&m=dev
3. プロフィール画面: https://figma.com/...?node-id=488-473&m=dev
```

**メリット**：
- 関連する画面を一度に実装できる
- デザインの一貫性が保たれる

## デザイン共有のコツ（うまくいく方法）

### ✅ 良い例

#### 1. URLに `&m=dev` を付ける
```
✅ 良い: https://figma.com/design/...?node-id=488-473&m=dev
❌ 悪い: https://figma.com/design/...?node-id=488-473
```
**理由**: Dev Modeの詳細情報も取得できる

#### 2. 具体的な画面名を伝える
```
✅ 良い: 「出店者プロフィール確認画面を実装してください」
❌ 悪い: 「これを実装して」
```
**理由**: 実装の意図がわかりやすい

#### 3. 複数画面はリストで
```
✅ 良い:
1. ログイン画面: [URL]
2. 登録画面: [URL]
3. プロフィール画面: [URL]

❌ 悪い: [URL1] [URL2] [URL3]（どれがどれかわからない）
```

### ❌ 避けるべきこと

1. **URLにnode-idがない**
   ```
   ❌ https://figma.com/design/... （node-idなし）
   ```
   → どの要素を実装すればいいかわからない

2. **複数の画面を一度に大量に**
   ```
   ❌ 50画面を一度に実装して
   ```
   → 1画面ずつ、または関連する数画面ずつが確実

## 実装プロセスの改善点

### 現在の問題点

1. **仕様書が長すぎる** → 全てを一度に確認しにくい
2. **実装漏れ** → グラデーションなど重要なスタイルを見落としやすい
3. **確認方法がない** → 実装が正しいか比較できない

### 改善策

#### 1. スクリーンショットを必ず取得

```
Figma URL → スクリーンショット取得 → 実装 → 比較
```

**メリット**：
- 視覚的に確認できる
- 違いがすぐわかる

#### 2. 実装チェックリストを使用

仕様書を「読む」のではなく、「チェックリスト」として使う：

```
【背景】セクションだけ確認
【ヘッダー】セクションだけ確認
【ボタン】セクションだけ確認
```

#### 3. 項目別に実装

```
1. 背景を実装 → スクリーンショットと比較
2. ヘッダーを実装 → スクリーンショットと比較
3. コンテンツを実装 → スクリーンショットと比較
```

## 完全実装のためのワークフロー

### 推奨ワークフロー

```
ステップ1: Figma URLを提供
   ↓
ステップ2: 私が以下を自動取得
   - デザインコンテキスト（get_design_context）
   - スクリーンショット（get_screenshot）
   - メタデータ（get_metadata）
   ↓
ステップ3: デザイン仕様書を生成
   - 必要な情報を整理
   ↓
ステップ4: 実装
   - スクリーンショットを見ながら実装
   - 仕様書を「辞書」として使用
   ↓
ステップ5: 検証
   - スクリーンショットと実装画面を比較
   - 違いがあれば修正
```
