# スマホブラウザUI考慮デザインガイド

## サービス名
**デミセル**（仮名）

---

## スマホブラウザの表示領域について

### 実際に使える領域

スマホのブラウザでは、以下のUI要素が表示領域を占有します：

1. **アドレスバー（上部）**
   - 高さ: 約44-56px
   - スクロール時に自動的に非表示になる（iOS Safari）

2. **ツールバー（下部、iOS Safari）**
   - 高さ: 約44px
   - スクロール時に自動的に非表示になる

3. **Safe Area（ノッチ・ホームインジケーター）**
   - iPhone X以降: 上部約44px、下部約34px
   - コンテンツが隠れないように余白を確保

---

## デザイン時の計算方法

### 方法1: Safe Areaを考慮した設計（推奨）

#### iPhone 16の場合
- **画面サイズ**: 393px × 852px（ポイント）
- **Safe Area**: 
  - 上部: 約59px（ステータスバー + ノッチ）
  - 下部: 約34px（ホームインジケーター）
- **実際のコンテンツ領域**: 393px × 約759px（852 - 59 - 34）

#### ブラウザUIを考慮
- **アドレスバー表示時**: さらに約56px減
- **実際に使える領域**: 393px × 約703px（759 - 56）

### 方法2: スクロール可能なコンテンツとして設計（推奨）

重要なコンテンツは、**最初の表示領域（Above the fold）**に配置：

- **最初の表示領域**: 約600-700px（高さ）
- **その他のコンテンツ**: スクロールで表示

---

## Figmaでの設定方法

### ステップ1: フレームサイズの設定

#### iPhone 16（実際の表示領域を考慮）
- **幅**: `393px`
- **高さ**: `852px`（フルスクリーン）
- **または**: `700px`（実際に使える領域を想定）

#### 推奨: フルスクリーンで設計
- フレームサイズ: `393px × 852px`
- 重要なコンテンツは上部 `700px` 以内に配置
- その他のコンテンツはスクロール可能として設計

---

### ステップ2: Safe Areaの設定

1. FigmaのiPhone 16テンプレートを使用している場合、Safe Areaは自動的に考慮されています
2. 手動で設定する場合：
   - 上部マージン: `59px`（ステータスバー + ノッチ）
   - 下部マージン: `34px`（ホームインジケーター）

---

### ステップ3: コンテンツの配置

#### 重要なコンテンツ（Above the fold）
- ロゴ
- タイトル
- 主要なボタン（ログインボタンなど）
- これらは**上部700px以内**に配置

#### スクロール可能なコンテンツ
- 説明文
- 追加情報
- フッター
- これらはスクロールで表示されることを想定

## WelcomeScreenの推奨レイアウト

### iPhone 16版

```
┌─────────────────────┐
│  Safe Area (59px)   │ ← ステータスバー + ノッチ
├─────────────────────┤
│                     │
│   ロゴ (160x48)     │ ← 上から64px
│                     │
│   タイトル          │ ← ロゴの下24px
│                     │
│  [LINEでログイン]   │ ← タイトルの下32px
│  [Googleでログイン] │ ← 16px間隔
│  [メールでログイン] │ ← 16px間隔
│                     │
│  新規登録           │ ← ボタンの下24px
│                     │
│  (スクロール可能)   │
│                     │
├─────────────────────┤
│  Safe Area (34px)   │ ← ホームインジケーター
└─────────────────────┘
```

### 実際の表示領域の計算

- **フルスクリーン**: 852px
- **Safe Area（上）**: 59px
- **Safe Area（下）**: 34px
- **アドレスバー（表示時）**: 56px
- **実際に使える領域**: 約703px（852 - 59 - 34 - 56）

**推奨**: 重要なコンテンツは**上部700px以内**に配置

## 実装時の考慮事項

### CSSでの実装

```css
/* Safe Areaを考慮 */
.container {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
}

/* ビューポートの高さを考慮 */
.min-h-screen {
  min-height: 100vh;
  /* または */
  min-height: 100dvh; /* Dynamic Viewport Height（推奨） */
}
```

### Next.jsでの実装

```tsx
<div style={{ 
  minHeight: '100dvh', // Dynamic Viewport Height
  paddingTop: 'env(safe-area-inset-top, 0px)',
  paddingBottom: 'env(safe-area-inset-bottom, 0px)'
}}>
  {/* コンテンツ */}
</div>
```

## まとめ

### デザイン時の推奨事項

1. **フルスクリーンで設計**（393px × 852px）
2. **重要なコンテンツは上部700px以内**に配置
3. **スクロール可能なコンテンツ**として設計
4. **Safe Areaを考慮**（Figmaテンプレートを使用している場合は自動）

### 実装時の推奨事項

1. **`100dvh`を使用**（Dynamic Viewport Height）
2. **`env(safe-area-inset-*)`を使用**（Safe Area対応）
3. **スクロール可能なコンテンツ**として実装
