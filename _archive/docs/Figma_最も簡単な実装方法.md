# Figma完全再現：最も簡単な実装方法

## この方法の特徴

✅ **視覚的に確認できる** - スクリーンショットで比較
✅ **漏れを防げる** - 見た目で違いがすぐわかる
✅ **ストレスが少ない** - 仕様書を読まなくてもいい

---

## ステップ1: Figma URLを提供

```
「この画面を実装してください」
https://figma.com/design/...?node-id=488-473
```

## ステップ2: スクリーンショットを取得して保存

Figmaのスクリーンショットを取得して、実装の参考にします。

## ステップ3: 実装後に比較

実装した画面とFigmaのスクリーンショットを並べて比較します。

## 実装時のコツ

### 1. 背景から実装

```
1. 背景色・グラデーションを確認
2. 実装
3. スクリーンショットと比較
```

### 2. 大きな要素から順に

```
1. ヘッダー（背景、タイトル、ボタン）
2. メインコンテンツ（カード、フォーム）
3. ボタン（色、サイズ、テキスト）
4. 細かいスタイル（シャドウ、角丸）
```

### 3. 1つずつ確認

各要素を実装したら、スクリーンショットと見比べて：
- 色が合っているか
- サイズが合っているか
- フォントが合っているか

## よくある問題と解決法

### 問題1: グラデーションが単色になっている

**解決法**: 仕様書の「背景」セクションを開いて、`linear-gradient(...)`をコピー

```tsx
// 仕様書からコピー
background: 'linear-gradient(-44.94deg, rgba(255, 245, 240, 1) 0%, rgba(232, 245, 245, 1) 99.95%)'
```

### 問題2: フォントが違う

**解決法**: 仕様書の「フォント」セクションを確認

```tsx
// 仕様書から
fontFamily: '"Inter", "Noto Sans JP", sans-serif'
fontSize: '18px'
fontWeight: 700  // Bold
fontStyle: 'italic'  // Italic（仕様書に書いてあれば）
```

### 問題3: シャドウがない

**解決法**: 仕様書の「シャドウ」セクションを確認

```tsx
// 仕様書からコピー
boxShadow: '0px 2px 8px rgba(0, 0, 0, 0.08)'
```

## 実用的なワークフロー

### 方法A: スクリーンショット比較（最も簡単）

```
1. Figma URLを提供
   ↓
2. スクリーンショットを取得して保存
   ↓
3. 実装しながらスクリーンショットを見る
   ↓
4. 実装後に比較して違いを修正
```

### 方法B: 仕様書を項目別に確認

実装する要素ごとに、仕様書の該当セクションだけを見る：

```
【背景を実装する】
→ 仕様書の「背景」セクションだけ見る

【ヘッダーを実装する】
→ 仕様書の「ヘッダー」セクションだけ見る

【ボタンを実装する】
→ 仕様書の「ボタン」セクションだけ見る
```

**仕様書全体を読む必要はない！**

## チェックポイント（実装時）

各要素を実装するときに、以下だけ確認：

### 背景
- [ ] 色コード（`#FFFFFF` など）
- [ ] グラデーション（`linear-gradient(...)` など）

### テキスト
- [ ] フォントサイズ（`18px` など）
- [ ] フォントウェイト（`700` = Bold, `400` = Regular）
- [ ] フォントスタイル（`'italic'` か `'normal'`）
- [ ] 色（`#2C3E50` など）

### ボタン・カード
- [ ] サイズ（`width: '289px'`, `height: '52px'`）
- [ ] 背景色
- [ ] 角丸（`borderRadius: '12px'`）
- [ ] シャドウ（`boxShadow: '...'`）
