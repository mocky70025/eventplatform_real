# Figma完全再現実装ガイド

## 問題：なぜ正確に実装できないのか？

1. **情報が多すぎる** - 仕様書は長く、全てを一度に確認しにくい
2. **実装漏れ** - 重要なスタイル（グラデーション、シャドウなど）を見落としやすい
3. **確認方法がない** - 実装が正しいか、Figmaと比較する方法がない

## 解決策：3ステップ完全再現法

### ステップ1: Figmaからスクリーンショットを取得して比較

Figma URLを提供すると、スクリーンショットを取得できます。

```
1. Figma URLを提供
2. スクリーンショットを取得
3. 実装後の画面と比較
```

### ステップ2: 仕様書を項目別にチェック

仕様書を読むのではなく、**実装チェックリスト**として使う：

#### チェックリスト項目

- [ ] **背景色・グラデーション**
  - 仕様書の「背景」セクションを確認
  - `background: 'linear-gradient(...)'` または `background: '#色コード'`
  
- [ ] **フォント設定**
  - fontFamily（"Inter" など）
  - fontSize（px単位）
  - fontWeight（700 = Bold, 400 = Regular）
  - fontStyle（'italic' または 'normal'）
  - color（#2C3E50 など）

- [ ] **サイズ・位置**
  - width, height（px単位）
  - padding, margin
  - border-radius（角丸）

- [ ] **エフェクト**
  - boxShadow（シャドウ）
  - border（ボーダー）

- [ ] **レイアウト**
  - display（'flex' など）
  - alignItems, justifyContent
  - flexDirection

### ステップ3: 実装後にスクリーンショットで検証

実装後、Figmaのスクリーンショットと実装画面を並べて比較：

```
Figma（スクリーンショット） ←→ 実装画面（ブラウザ）
```

目視で違いを確認し、修正する。

## 具体的な実装フロー

```
1. Figma URLを提供
   ↓
2. スクリーンショットを取得（get_screenshot）
   ↓
3. 仕様書を確認しながら実装
   ↓
4. 実装後にスクリーンショットと比較
   ↓
5. 違いがあれば修正
```

### 方法2: 項目別チェック法

```
1. 背景を実装
   - 仕様書の「背景」セクションを確認
   - グラデーションか単色か
   - 色コードを正確に

2. ヘッダーを実装
   - 背景色、高さ、タイトルテキスト
   - フォントサイズ、色、ウェイト

3. 各要素を順番に実装
   - カード、ボタン、入力欄など
   - 各要素のスタイルを仕様書で確認
```

## よくある実装漏れと対処法

### 1. グラデーションが反映されない

**問題**: 仕様書には書いてあるが、実装で単色になっている

**対処法**:
```tsx
// ❌ 間違い
background: '#E8F5F5'

// ✅ 正しい（仕様書からコピー）
background: 'linear-gradient(-44.94deg, rgba(255, 245, 240, 1) 0%, rgba(232, 245, 245, 1) 99.95%)'
```

**チェック**: 仕様書の「背景」セクションで「グラデーション」と書いてあれば、linear-gradientを使う

### 2. フォントスタイルが違う

**問題**: Bold ItalicがRegularになっている

**対処法**:
```tsx
// ❌ 間違い
fontWeight: 400,  // Regular

// ✅ 正しい（仕様書から）
fontWeight: 700,  // Bold
fontStyle: 'italic'  // Italic
```

**チェック**: 仕様書のフォントセクションで「Bold Italic」と書いてあれば、fontWeight: 700 と fontStyle: 'italic' の両方

### 3. シャドウが抜けている

**問題**: カードやボタンにシャドウがない

**対処法**:
```tsx
// 仕様書の「シャドウ」セクションからコピー
boxShadow: '0px 2px 8px rgba(0, 0, 0, 0.08)'
```

## 実践的なワークフロー

### 1画面を実装する手順

```
1. Figma URLを提供
   ↓
2. 仕様書を生成（自動）
   ↓
3. スクリーンショットを取得（視覚的確認用）
   ↓

   ↓
```

### チェックポイント（実装中）

実装するたびに、以下を仕様書で確認：

1. **背景は？** → 仕様書の「背景」セクション
2. **このテキストのフォントは？** → 仕様書の「フォント」セクション
3. **このボタンの色は？** → 仕様書の「ボタン」セクション
4. **シャドウはある？** → 仕様書の「シャドウ」セクション
